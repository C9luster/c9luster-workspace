# Redis 基础知识

## 1. 数据结构

### 1.1 基础类型

| 类型 | 底层实现 | 常用命令 | 典型场景 |
|------|----------|----------|----------|
| String | SDS | GET/SET/INCR/DECR | 缓存、计数器、分布式锁 |
| Hash | 压缩列表/哈希表 | HGET/HSET/HGETALL | 对象存储、购物车 |
| List | 快速链表 | LPUSH/RPOP/LRANGE | 消息队列、最新列表 |
| Set | 整数集合/哈希表 | SADD/SMEMBERS/SINTER | 标签、共同好友、去重 |
| ZSet | 跳表+哈希表 | ZADD/ZRANGE/ZRANK | 排行榜、延迟队列 |
| Stream | 基数树 | XADD/XREAD/XGROUP | 消息队列、消费组 |

### 1.2 高级类型

- **Bitmap**：位图，用于签到、在线状态、布隆过滤器
- **HyperLogLog**：基数统计，用于 UV 统计（误差约 0.81%）
- **GEO**：地理位置，用于附近的人、距离计算

## 2. 持久化机制

### 2.1 RDB（快照）

- **原理**：定时将内存数据快照写入磁盘
- **优点**：恢复快、文件紧凑
- **缺点**：可能丢失最后一次快照后的数据
- **配置**：`save 900 1`（900 秒内至少 1 次修改则触发）

### 2.2 AOF（追加日志）

- **原理**：记录每条写命令，重启时重放
- **优点**：数据更安全，可配置 fsync 策略
- **缺点**：文件大、恢复慢
- **重写**：`BGREWRITEAOF` 压缩 AOF 文件

### 2.3 混合持久化（Redis 4.0+）

- RDB 做全量备份 + AOF 做增量，兼顾恢复速度与数据安全

## 3. 高可用架构

### 3.1 主从复制

- 异步复制，从节点可读
- 全量同步（RDB）+ 增量同步（repl_backlog）

### 3.2 哨兵（Sentinel）

- 监控、自动故障转移、配置提供
- 至少 3 个哨兵节点，多数派选举

### 3.3 Cluster 集群

- 16384 个槽位分片，无中心节点
- 支持水平扩展，自动故障转移

## 4. 缓存策略

### 4.1 常见模式

- **Cache Aside**：先更新 DB，再删缓存（常用）
- **Write Through**：同时写缓存和 DB
- **Write Behind**：先写缓存，异步写 DB

### 4.2 缓存问题

- **穿透**：查询不存在的数据 → 布隆过滤器、缓存空值
- **击穿**：热点 key 过期 → 互斥锁、逻辑过期
- **雪崩**：大量 key 同时过期 → 过期时间加随机值、多级缓存

## 5. 分布式锁

```redis
# 加锁（需保证原子性）
SET key value NX EX 30

# 释放锁（Lua 脚本保证原子性，校验 value 防止误删）
if redis.call("get", KEYS[1]) == ARGV[1] then
    return redis.call("del", KEYS[1])
else
    return 0
end
```

- **Redisson**：看门狗自动续期
- **Redlock**：多实例加锁，多数派成功才算加锁成功

## 6. 性能优化要点

- 使用 Pipeline 批量操作
- 避免大 key（单 key 超过 10KB 需警惕）
- 合理设置 maxmemory 和淘汰策略
- 使用连接池
- 热点 key 拆分或本地缓存
