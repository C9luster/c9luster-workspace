# Vue 基础学习（Vue 3）

本文面向 Vue 3（组合式 API 为主），覆盖日常开发最常用的概念与实践要点。目标是：看懂项目代码、能独立写页面、理解组件通信与状态管理的基本套路。

## 1. Vue 是什么

- **定位**：用于构建用户界面的渐进式框架。
- **核心能力**：声明式渲染、组件化、响应式状态、完善的生态（Router/Pinia/Vite）。

## 2. 推荐开发栈（新项目）

- **构建工具**：Vite
- **语言**：TypeScript（建议）
- **路由**：Vue Router
- **状态管理**：Pinia
- **代码规范**：ESLint + Prettier（按团队落地）

## 3. 单文件组件（SFC）

Vue 最常见的文件形态是 `.vue` 单文件组件，通常包含三块：

- **`<template>`**：模板（UI 结构）
- **`<script setup>`**：逻辑（状态、函数、依赖）
- **`<style scoped>`**：样式（可选，`scoped` 表示样式作用域隔离）

## 4. 模板语法速览

- **插值**：`{{ msg }}`
- **属性绑定**：`:class="cls"`、`:disabled="loading"`
- **事件绑定**：`@click="onClick"`
- **双向绑定**：`v-model="value"`（常见于表单）
- **条件渲染**：`v-if / v-else-if / v-else`
- **列表渲染**：`v-for="item in items"`（建议同时写 `:key`）

实践建议：

- **`v-if` vs `v-show`**：`v-if` 会创建/销毁节点；`v-show` 只切换 `display`。频繁切换可优先 `v-show`。
- **`key` 的选择**：优先用稳定唯一 ID，避免用数组下标（除非列表不会重排/插入）。

## 5. 响应式：`ref`、`reactive`

Vue 3 的响应式基于 Proxy + 依赖追踪：

- **`ref`**：适合基本类型或“需要整体替换”的值；在 `<script setup>` 内通过 `.value` 访问，在模板中自动解包。
- **`reactive`**：适合对象/数组；更像“就地可变”的状态容器。

实践建议：

- 在组件边界与公共 API 上，尽量让数据结构清晰（配合 TS 类型）。
- 避免把大量状态无节制放在全局 store；能局部就局部。

## 6. 计算属性与侦听：`computed`、`watch`

- **`computed`**：从现有状态派生新值；有缓存，依赖不变不会重复计算。适合“展示层派生”。
- **`watch`**：监听变化执行副作用；适合请求、日志、与外部系统同步等“副作用逻辑”。

选择建议：

- **能用 `computed` 就别用 `watch`**（尤其是仅做值转换/拼接的场景）。
- `watch` 默认是懒执行（初始不会触发），需要立即执行用 `immediate: true`。

## 7. 生命周期（组合式 API）

常见生命周期钩子：

- `onMounted`：组件挂载后（可请求数据/订阅）
- `onUnmounted`：组件卸载前后（清理定时器/取消订阅）

实践建议：

- 所有“注册型副作用”（事件监听、定时器、订阅）都要对应清理，避免泄漏。

## 8. 组件通信：Props / Emits / Slot

- **Props（父 -> 子）**：父组件把数据/回调传入子组件；子组件只读使用 props（不要直接改）。
- **Emits（子 -> 父）**：子组件通过事件通知父组件（表单提交、点击选择等）。
- **Slot（内容分发）**：父组件把一段模板/内容传给子组件渲染；适合容器组件、布局组件。

实践建议：

- **props 下发数据，emit 上抛意图**：子组件尽量“无业务决策”，只表达用户意图与交互结果。
- 组件设计优先“窄接口”：props 少而清晰，避免传入大对象后在内部随意取字段。

## 9. 组合式函数（Composables）

把可复用逻辑抽到 `useXxx()` 函数中（例如 `useFetch`、`usePagination`、`useDebounce`），典型收益：

- 逻辑复用、测试更容易
- 组件变轻，关注点更清晰

建议：

- Composable 内尽量只做“通用能力”，避免耦合具体页面/路由。
- 对外暴露的返回值尽量稳定、可读（状态 + 动作）。

## 10. 路由（Vue Router）

常见概念：

- 路由表：path -> component
- 动态路由：`/user/:id`
- 路由守卫：登录态、权限控制、埋点

实践建议：

- 权限校验集中在守卫/中间层，页面只处理“展示与交互”。
- 路由参数与查询参数要有明确的解析与类型约束（TS + 校验）。

## 11. 状态管理（Pinia）

适用场景：

- 多页面共享状态（用户信息、主题、权限、全局配置）
- 需要跨组件树共享并可追踪更新

实践建议：

- store 里放“可复用的业务状态”，不要把所有页面临时状态都塞进去。
- 异步请求建议封装成 actions，并在 UI 层处理 loading/error 展示策略。

## 12. 表单与 `v-model`

常见模式：

- 原生输入控件：`v-model`
- 自定义组件：通过 `modelValue` + `update:modelValue` 形成可组合的双向绑定协议

实践建议：

- 表单校验要么采用成熟方案（如 VeeValidate），要么在业务层明确校验时机（提交前/实时/失焦）。
- 避免把“显示值”和“提交值”混在一起（尤其是格式化金额/日期）。

## 13. 性能与体验基础

- **减少不必要渲染**：大列表使用虚拟列表；避免在模板里写重计算逻辑（用 `computed`）。
- **拆分组件**：把大组件拆成职责清晰的子组件，减少耦合与重渲染范围。
- **按需加载**：路由懒加载、组件异步加载。

## 14. 常见坑（高频）

- **把响应式对象解构后丢失响应性**：从 `reactive` 解构需要用 `toRefs`（或直接用 `ref`）。
- **在模板里滥用函数调用**：会在每次渲染都执行；能用 `computed` 就用 `computed`。
- **列表 key 不稳定**：导致复用错乱、状态错位。
- **副作用不清理**：计时器/事件监听/订阅没有在卸载时清理。

## 15. 学习路径建议

- **先会写页面**：模板语法 + `ref/reactive` + `computed/watch` + 组件拆分
- **再学工程化**：路由、Pinia、请求封装、错误处理、权限与布局
- **最后补“可维护性”**：组件设计、组合式函数抽象、性能与测试
